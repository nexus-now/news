STEP 04
Twitter / X API (4 Keys)
The "Save Changes" button problem — here's the exact fix.
⏱ 20 MINUTES
WHY 4 KEYS?
Twitter uses 4 different codes to verify your app: API Key (who you are), API Secret (your app's password), Access Token (permission to post), Access Secret (token's password). You need all 4. The "Save changes greyed out" is a common problem with a specific fix below.
1
Go to Twitter Developer Portal
OPEN THIS URL — sign in with your NEXUS NOW Twitter accounthttps://developer.twitter.com/en/portal/dashboard
2
Create a Project and App

Click "Create Project". Give it a name like NEXUS NOW. Select use case: "Making a bot" or "Publishing and curating tweets". Finish the wizard.

Then click "Create App" inside the project. Name it nexusnow-publisher.

3
Fix the "Save Changes" greyed out problem

This happens because Twitter requires you to add a Callback URL before they let you save Read+Write permissions. Here is the fix:

Your App page → ⚙️ Settings tab → User authentication settings → Click "Set up"

Fill in the User Authentication form:

App permissions — select thisRead and Write
Type of App — select thisWeb App, Automated App or Bot
Callback URI (Required — paste this exactly)https://nexus-now.github.io/nexus-now/callback
Website URL (Required)https://nexus-now.github.io/nexus-now

Now click Save. The button will no longer be greyed out because you filled in the required URL fields.

4
Get your 4 keys
Your App → "Keys and Tokens" tab

You will see sections. Here's what to copy from each:

Section "Consumer Keys" — copy "API Key" → GitHub Secret name:TWITTER_API_KEY
Section "Consumer Keys" — copy "API Key Secret" → GitHub Secret name:TWITTER_API_SECRET

Now click "Generate" under the "Access Token and Secret" section:

Copy "Access Token" → GitHub Secret name:TWITTER_ACCESS_TOKEN
Copy "Access Token Secret" → GitHub Secret name:TWITTER_ACCESS_SECRET
⚠️ Access Token and Secret also only show ONCE. Copy both immediately before closing.
Make sure it says "Created with Read and Write permissions" under the Access Token. If it says Read Only, delete it, go back to set up permissions (Step 3), save, then regenerate.

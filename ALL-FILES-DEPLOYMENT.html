Part 2 — Every File Explained

What each file does, in plain English.

ROOT FILES (go directly into the repo, no subfolder)

FILE NAMETYPEWHAT IT DOES
index.htmlWEBSITEThe public-facing NEXUS NOW website. Live news ticker, hero section, latest stories grid, AI team showcase, pipeline diagram. Automatically updated with new articles via news.json.
dashboard.htmlWEBSITEMission Control panel. Shows all 11 agents online, quality scores, pipeline live log, platform publishing status.
training-dashboard.htmlWEBSITETraining Intelligence dashboard. Shows all 5 self-training layers for every agent — samples collected, wisdom rules learned, cross-agent pool, adversarial rewrites.
setup-guide.htmlGUIDEOriginal setup guide with architecture diagrams. Can keep for reference.
SETUP-GUIDE-COMPLETE.htmlGUIDEFull zero-knowledge step-by-step setup guide for every platform.
news.jsonDATAStarts empty (just "[]"). Every time the pipeline runs, it adds the new articles here. The website reads this file to show latest stories. Auto-trimmed to 100 articles max.
orchestrator.pyCODEThe MASTER brain. GitHub Actions calls this file twice daily. It coordinates all 11 agents: SCOUT finds trends → specialist agent writes → AEGIS verifies → HERALD publishes → training records everything.
pipeline.pyCODEOriginal simpler pipeline from v1. Kept as backup reference. Not actively used — orchestrator.py is the current one.
requirements.txtCONFIGList of Python libraries the code needs. GitHub Actions reads this and installs them automatically before every run. You never touch this file.
README.mdCONFIGDescription shown on your GitHub repo page. Explains what NEXUS NOW is.
.gitignoreCONFIGTells GitHub which files to ignore. Prevents large media files and secret files from being accidentally uploaded.

agents/ FOLDER (the intelligence)

FILE NAMEAGENTWHAT IT DOES
__init__.py—Empty file. Makes the agents/ folder recognisable as a Python package. Required. Don't delete it.
base_agent.pyBASE CLASSThe parent brain all agents inherit from. Contains: memory loading/saving, prompt evolution, self-evaluation, performance tracking, and the generate_with_training() method that fires all 5 training layers.
free_ai_provider.pyAI ROUTER8-provider failover engine. Tries Gemini → Groq → Together → Cohere → HuggingFace → Mistral → OpenRouter in order. Also handles image generation (Pollinations.ai free) and TTS (Edge TTS free). System never stops when one provider fails.
training_engine.pyTRAINERAll 5 self-training layers: (1) Experience Memory JSONL store, (2) Few-shot injection, (3) Pattern extraction every 3 runs, (4) Cross-agent wisdom pool, (5) Adversarial self-challenge rewriter. Also exports HuggingFace datasets and generates Google Colab fine-tuning scripts.
scout_agent.pySCOUT 👁️Fetches Google Trends RSS (free, no key needed). Classifies each trend into a category. Scores them. Selects the top 2. Routes each to the correct specialist agent.
category_agents.py8 AGENTSContains all 8 specialist category agents in one file: TITAN (Business), PULSE (Science), VOLT (Technology), ARENA (Sports), NEXUS (World Politics), PRISM (Entertainment), TERRA (Environment), CIPHER (Crime). Each has deep domain expertise and its own training loop.
aegis_agent.pyAEGIS 🛡️Fact-checks every story before publishing. Assigns VERIFIED/DEVELOPING/UNVERIFIED/BLOCKED status. Generates unique copyright ID (e.g. NN-2026-0404-143022-TEC-A7F3). Stamps the ID into YouTube description, Instagram caption, tweets, video script, and website JSON-LD. Blocks unsafe content.
herald_agent.pyHERALD 📡The publisher. Generates thumbnail image (Gemini/Pollinations), voiceover audio (Google TTS/Edge TTS), posts to Twitter thread, Instagram, YouTube, updates news.json for website, uploads media to Google Drive.
storage_manager.pySTORAGEManages all free Google Workspace storage. Saves articles to Google Sheets. Logs pipeline to Sheets (capped at 500 rows). Uploads media to Drive with auto-expiry dates. Runs cleanup at pipeline start. Monitors Drive usage — alerts if approaching 15GB limit.

Part 4 — Exact Upload Instructions
1
Download all files from Claude to your computer

Every file I've given you in this chat has a download button (the arrow icon at the top right of each file card). Click it for each file. Save them all to a single folder on your computer — name it nexus-now.

Make sure you recreate the folder structure: create an agents subfolder, put all the agent .py files inside it. Create a brand subfolder, put all logo files inside it. Create a .github/workflows subfolder, put pipeline.yml inside it.

The .github folder starts with a dot — on Windows/Mac it may be hidden. Make sure it's included. The pipeline.yml file MUST be at exactly: .github/workflows/pipeline.yml
2
Go to your GitHub repository

Open your browser. Go to:

https://github.com/nexus-now/nexus-now

Sign in if asked. You should see your empty repository (just a README.md).

3
Upload the root files first
Your repo page→ Add file button→ Upload files

Drag and drop these files from your nexus-now folder (NOT from any subfolder — just the root files):

index.html dashboard.html training-dashboard.html setup-guide.html SETUP-GUIDE-COMPLETE.html news.json orchestrator.py pipeline.py requirements.txt README.md .gitignore

At the bottom of the page, in the Commit changes box, type: Initial upload - root files. Click the green Commit changes button.

4
Upload the agents/ folder

GitHub's web interface can't create folders directly. Use this trick:

Add file→ Upload files→ Drag the entire "agents" folder→ GitHub will detect the folder structure

If dragging a folder doesn't work, click "Add file" → "Create new file", then type the file path as: agents/__init__.py — GitHub will automatically create the agents/ folder. Repeat for each file:

agents/__init__.py (empty file - no content needed) agents/base_agent.py agents/free_ai_provider.py agents/training_engine.py agents/scout_agent.py agents/category_agents.py agents/aegis_agent.py agents/herald_agent.py agents/storage_manager.py

For each: click Add file → Create new file → type the path including "agents/" prefix → paste the file content → Commit.

5
Upload the .github/workflows/pipeline.yml file

This is the most important file — it's what makes everything run automatically.

Add file→ Create new file→ In name box type: .github/workflows/pipeline.yml→ Paste the content→ Commit
When you type .github/workflows/pipeline.yml in the name box, GitHub automatically creates the nested folders. You'll see the path appear above the text area.
6
Upload the brand/ folder and placeholder folders

Upload all brand files the same way — using "Create new file" with path like brand/logo-dark.png. But wait — PNG files can't be pasted as text. For images, use "Upload files" and drag them while you're inside the brand/ folder.

Create placeholder files for the empty folders:

Add file → Create new file→ Name: agent_memory/.gitkeep→ Leave content empty→ Commit

Repeat for: assets/images/.gitkeep, assets/audio/.gitkeep, logs/.gitkeep

7
Enable GitHub Pages
Settings tab→ Pages (left sidebar)→ Source: Deploy from a branch→ Branch: main→ Folder: / (root)→ Save

Wait 2 minutes. Your website is now live at:

https://nexus-now.github.io/nexus-now/
8
Add all GitHub Secrets
Settings tab→ Secrets and variables→ Actions→ New repository secret

Add these one at a time. The minimum set to get started:

Name: GEMINI_API_KEY ← your new Gemini key from aistudio.google.com Name: GITHUB_USERNAME ← type exactly: nexus-now Name: GROQ_API_KEY ← from console.groq.com → API Keys → Create

Add the rest as you complete each platform setup (see SETUP-GUIDE-COMPLETE.html).

9
Upload brand assets to social platforms (manual, one-time)
PLATFORMFILE TO UPLOADWHERE IN THE PLATFORM
📺 YouTubeicon-512.pngYouTube Studio → Customisation → Branding → Profile picture
📺 YouTubelogo-dark.pngYouTube Studio → Customisation → Branding → Banner image
📸 Instagramicon-512.pngProfile → Edit Profile → Change profile photo
🐦 Twitter/Xicon-512.pngProfile → Edit Profile → Avatar (circle image)
💼 LinkedInicon-512.pngCompany Page → Edit → Logo
💼 LinkedInlogo-light.pngCompany Page → Edit → Cover image
🌐 Websitefavicon.icoAlready in repo → auto-shown in browser tab
10
Test the pipeline manually
Actions tab→ Left sidebar: NEXUS NOW Pipeline→ Run workflow button (right side)→ Run workflow

Watch the yellow spinning icons. They turn green ✅ when each step succeeds. Click into the run to see detailed logs. Any ❌ red icon means an error — click on it and read the error message to know which secret is missing or wrong.

Once it runs green, NEXUS NOW is fully live. The pipeline now runs forever, automatically, twice a day, with zero human intervention. All 11 agents are live, self-training, and getting smarter with every article.
